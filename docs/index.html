MMIO++

Macro-familiar register code, without macro-era type holes.

MMIO++ keeps the shape embedded developers already know: register names, field names, encoded values, and masks. The difference is that the compiler now enforces which pieces may be combined, assigned, cleared, compared, or rejected.

Macro-Familiar API Verification Story

The Project Story

Embedded register code often starts from C macros because they are direct, terse, and map cleanly onto datasheets. MMIO++ does not fight that mental model. It refines it.

A field still has named states and a derived mask. A register still feels like a fixed address. The public API keeps those concepts distinct so the compiler can block the classic mistakes: mixing fields from different registers, writing masks as values, or using value fields in bitwise-clear paths.

What The Repo Proves

  • Header-only core abstractions live in include/mmio.hpp.
  • Example register maps stay outside the framework header.
  • Host checks validate API shape and compile-fail misuse cases.
  • QEMU runtime tests execute register code at real target addresses.
  • Cortex-R5 runs under the same Windows-first preset flow as the other test paths.

Why The API Looks Familiar

Classic macro style

SPI_CR = SPI_CR_SPIEN_ENABLE
       | SPI_CR_SWRST_RESET;

SPI_CR |= SPI_CR_SPIEN_ENABLE;
SPI_CR &= ~SPI_CR_SWRST_Msk;

SPI_MR = SPI_MR_MSTR_MASTER
       | SPI_MR_DLY(7);

MMIO++ style

SPI_CR::Instance<0xFFFE0000u> spiCr;
SPI_MR::Instance<0xFFFE0004u> spiMr;

spiCr = SPI_CR::SPIEN::ENABLE
      | SPI_CR::SWRST::RESET;

spiCr |= SPI_CR::SPIEN::ENABLE;
spiCr &= ~SPI_CR::SWRST::MASK;

spiMr = SPI_MR::MSTR::MASTER
      | SPI_MR::DLY::value(7);

The names stay close to register-map macros. The types change what the compiler is allowed to accept.

Values vs Masks

MMIO++ treats encoded values and masks as different public tokens because they mean different things in real register code.

  • FIELD::VALUE_NAME is for whole-register writes, predicates, and OR-safe set operations.
  • FIELD::MASK is for clear or toggle operations.
  • FIELD::value(x) is for numeric value fields.

Register Operations

spiMr.set<SPI_MR::MSTR::MASTER>();
spiMr.set(SPI_MR::MSTR::MASTER
        | SPI_MR::DLY::value(7));
spiMr.set<SPI_MR::DLY>(7);

const bool isMaster =
  (spiMr & SPI_MR::MSTR::MASTER);
